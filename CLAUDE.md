# OCHA QuickVid — instructions for Claude

Two products, one shared brand engine. Everything runs locally; **user video
never leaves the machine**.

1. **Web app** — FastAPI backend (`app/backend/`) + static SPA (`browser/`) +
   `engine/`. The page is the UI for a local engine on `127.0.0.1:17870`. Two
   modes: **Edit** (transcribe → cut → brand) and **Titles & branding** (brand a
   finished clip). Public at `un-ocha.github.io/quickvid_BDU`; pushes to `main`
   auto-deploy (GitHub Pages).
2. **Premiere plugin** — `premiere/cep/` (CEP panel; ExtendScript host in
   `jsx/host.jsx`) places OCHA MOGRTs. The MOGRTs are generated by the AE builder
   in `premiere/ae/` (edit `src/builder_template.jsx` → run `make_assets.py` →
   run the generated `build_ocha_mogrts.jsx` in After Effects). Ships as a signed
   `.zxp` in `distribution/`, auto-updates via `version.json`.

**The full why for any decision lives in `docs/decisions.md`** (the dated
logbook). This file is just the map + standing rules.

## What's where
- `engine/` — the render modules, shared by both web modes: `statement.py`
  (cut+brand orchestrator), `social_brand.py` (the ffmpeg overlay graph),
  `finish.py` (Titles path), `lower_third.py` · `pin_locator.py` · `text_on.py`
  (SVG→PNG element renderers), `mediakit.py` (the ONE colour gate: HDR/wide-gamut
  → bt709), `look.py`, `compress.py`, `ending.py`, `webtv.py`, `svgpng.py`.
- `browser/` — the SPA. `app.js` (Titles tab + engine gate), `statement.js`
  (Edit tab), and SHARED both-tab modules: `captions.js`, `look.js`,
  `texton.js`, `lowerthird.js`, `location.js`, `toolbox.js`.
- `browser/brand-lt.json` + `brand-pin.json` — the brand NUMBERS (single source
  of truth). The engine renders from them AND the AE builder bakes from them, so
  the plugin and the web app stay in step.
- `distribution/ocha_quickvid_plugin_download/` — the signed `.zxp` + Windows
  setup + PDF guide colleagues install from. Built by `tools/build-panel-package.sh`.

## Standing rules (these have bitten us — see decisions.md)
- **Kit-first UI.** Look & feel is the shared **OCHA App Kit**. Edit the KIT
  (`…/OCHA_design_system/ocha-common-design-system-BDU/app-kit/ocha-app-kit.css`),
  never the synced copy (`browser/vendor/ocha-app-kit.css`). Reuse a `cd-*`
  component before inventing; if truly new, add it to the kit. Then:
  `cd …/app-kit && python3 sync.py`, log `CHANGELOG.md`, and **remind Javier to
  prompt his Design System session** (a handoff is added automatically).
  `browser/style.css` is QuickVid LAYOUT only — no colours, no component styling.
- **`host.jsx` is ExtendScript (ES3): `var` only, no arrows/template-literals,
  ASCII-only, and never `--` inside a comment** (it breaks the manifest XML too).
- **Never rebuild/sign the `.zxp` without Javier's explicit go.** Stage
  everything; he authorises the release.
- **Restart the engine for backend (Python) changes** — the launch config has no
  `--reload` and static files are served no-cache.
- **Share, don't duplicate.** Both-tab logic lives in ONE module (the shared
  `browser/*.js`; `engine/` is shared by both modes). Copy-paste has drifted before.
- **Logos are always SVG**, rasterised at render time — never ship a PNG.
- **Versioning is CalVer** `2026.MINOR.PATCH`: for the plugin, `2026.0.x` =
  internal testing / `2026.1+` = public; the web app's `VERSION` continues the
  `2026.0.x` counter. Keep every version site in lockstep (manifest, panel const,
  cache-busts, `version.json`; web `VERSION` + `ENGINE_LATEST`).
